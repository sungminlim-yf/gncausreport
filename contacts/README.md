# contacts/ — 명함 인사이트 메모 (네트워크 컨텍스트)

이 폴더는 **businesscard 파이프라인**(`../businesscard/`)이 행사·미팅에서 받은 명함을
처리하며 자동 생성하는 인사이트 메모를 모은다. 각 파일 1개 = 만난 인물 1명.

## 파일 형식
- 파일명: `<YYYY-MM-DD>_<회사>_<이름>.md`
- 구조: frontmatter(name·company·title·email·date 등) + 본문
  - `## 인물·회사 요약`
  - `## GNC 호주 사업 관점` (시사점 / 기회 / 리스크 / 후속 액션)
- 본문은 `facts.md`·`context.md`를 grounding 으로 생성되어 회사 사실과 일관됨.

## 주간 보고서가 참조하는 법 (D-연계)
- `/brief` 의 **3단계 writer** 가, 주제가 네트워크·파트너·인물과 관련되면
  이 폴더의 최근 메모를 **내부 컨텍스트**로 읽어 반영한다(`facts.md` 와 동급 취급).
  반영 시 본문에 `(명함/네트워크 기반)` 으로 출처 성격을 표기.
- 주제가 **"신규 컨택 인사이트"** 이면 이 폴더의 최근 메모가 **주 입력**이 된다.
- `_recent-digest.md` 는 최근 메모를 한 파일로 모은 요약본(자동 갱신) — writer·Q&A 가
  개별 파일을 모두 열지 않고도 빠르게 참조하도록 제공.

## 운영 메모
- 개별 메모는 사람이 자유롭게 수정·보강해도 됨(다음 보고서에 반영).
- 민감·대외비는 `facts.md` 규칙과 동일하게 다룰 것(외부 채널 유출 주의).
