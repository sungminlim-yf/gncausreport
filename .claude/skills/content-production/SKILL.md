# content-production — 사실기반 재생산 양식

선별된 자료(`curator.json`)를 **사실에 기반해 논리적으로 재구성**하고, 한국어 독자가 쉽게 이해할 수 있게 가공한다. 결과는 `runs/<run-id>/draft.md`.

## 출력 구조 (고정)

```
# <제목>

## 핵심 요약 (3줄)
- <결론 1>
- <결론 2>
- <결론 3>

## 본문
<논리 구조화된 한국어 본문. 주장마다 [번호] 인용>

## 📎 출처
[1] <매체/기관> — <원문 제목> (<발행일>) <URL>
[2] ...
```

## 문체 (보고형·개조식) ★
- 모든 문장을 **보고형(개조식) 종결**로 쓴다: `~함`, `~임`, `~됨`, `~음`, `~필요`, 또는 명사형으로 끝맺는다.
- **금지**: `~이다 / ~한다 / ~된다 / ~했다 / ~입니다 / ~합니다` 같은 설명형·구어체 종결.
- 변환 예)
  - 시장이 빠르게 성장하고 있다. → 시장이 빠르게 성장 중임.
  - 발효일을 점검해야 한다. → 발효일 점검 필요.
  - GNC의 강점과 정확히 맞는다. → GNC 강점과 정확히 부합함.
  - 신규 공급자에게 진입 여지가 있다. → 신규 공급자에게 진입 여지 있음.
- **핵심요약·본문·`→` 시사점·`왜 중요한가` 모두** 동일하게 보고형 유지.
- 단, 수치·고유명사·인용구·URL은 문체 변환 대상이 아니므로 원문 그대로 둔다.

## 언어 규칙 (D3 — 한국어 본문 + 원문 병기)
- **본문은 한국어**로 쓴다(독자는 슬랙 한국어 사용자).
- **핵심 수치·고유명사·규제명·인용구는 원문(영어)을 괄호로 병기**한다.
  예) `호주 보조식품 시장은 2024년 약 58억 호주달러(AUD 5.8 billion) 규모로 성장함 [1].`
  예) `새 규정은 2026년 7월 발효 예정(Therapeutic Goods (Permissible Ingredients) Determination) [2].`
- 번역은 의미 왜곡 없이. 수치·날짜·단위는 원문 그대로 유지하고 한국식 표기를 병기.
- **영어는 고유명사·기관/제품/규제명·단위에만.** 일반 개념·보통명사는 **한국어로 풀어 쓴다** — 한국 경영진의 빠른 이해가 최우선. 영어 단독·약어 남용 금지.
  - 자주 나오는 교정 예: `prime power → 상시전원(주 전원)` · `YoY → 전년 대비` · `by product type → 제품 유형 기준` · `concurrence(agency) → 협의(동의) 절차` · `drop-in → 무개조 주입` · `backlog → 수주잔고` · `lead time → 납기` · `standby → 비상(대기) 전원`
  - 꼭 필요한 전문용어만 **'한국어(영어)' 순**으로 1회 병기하고, 이후엔 한국어만 쓴다. (고유명사·제품명은 영어 유지)

## 인용 규칙 (출처는 1급 시민)
- **모든 사실·수치·인용 주장에 `[번호]`를 붙인다.** 근거 없는 단정 금지.
- 번호는 📎출처 목록과 1:1 대응. 한 문장이 여러 출처면 `[1][3]`.
- reviewer가 고위험 주장(수치·규제 발효일 등)을 원문과 대조하므로(D18), 해당 주장의 출처를 정확히 매핑한다.

## 저작권·인용 원칙 (D12)
- **원문 장문 복제 금지.** 사실·수치 중심으로 **재서술**한다.
- 직접 인용은 **최단(한두 문장)**으로만, 따옴표와 출처를 명시.
- 항상 원문 링크 제공. 산출물은 **사내 참고용**임을 전제.

## 가독성
- 쉬운 표현, 짧은 문장. 전문용어는 처음 등장 시 풀어 설명.
- 결론 우선(역피라미드). 경영진은 3줄 요약에서 핵심을 얻는다.

## 길이·간결성 예산 (임팩트 우선)
> 슬랙에서 길면 임팩트가 떨어지고 스크롤 반응도 느려진다. **짧고 단단하게** 쓴다.
- **핵심 요약**: 정확히 3줄. 각 줄 **1문장(≤약 90자)**, 결론만. 수식어·배경 제거.
- **본문**: 섹션당 **2~4문장** + `→` 시사점 1줄. 같은 사실을 요약과 본문에서 반복하지 않는다.
- **전체 분량**: 슬랙 1~2 스크롤 내(대략 3,500자 이하 권장). 넘으면 **자른다 — 완전성보다 임팩트**.
- **절대 유지**: 핵심 수치·발효일·고유명사·`[번호]` 인용·📎출처. (줄이는 건 설명·배경이지 근거가 아니다)
- 군더더기(장황한 배경, 중복 설명, 당연한 일반론)는 과감히 삭제.
- **단, 압축이 의미를 해치면 안 된다.** 약어·단어 나열로 뜻이 모호해질 바엔 한 구절 더 써서 명확히 한다(완전성보다 임팩트이되, 임팩트보다 *이해*가 먼저).
- **경영진 체크포인트**: 각 항목을 **'무엇을 — 왜/어떻게'가 드러나는 완결된 한 문장**으로. 핵심어만 나열하는 과압축 금지.
  - 나쁨) `타깃: QLD` → 좋음) `1차 타깃을 퀸즐랜드(브리즈번 권역) 신규 데이터센터로 좁힘`
  - 나쁨) `규제: AEMC` → 좋음) `AEMC 계통연계 표준 확정 일정을 추적해 발전기 사양에 반영함`
