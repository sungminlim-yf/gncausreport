# curate — 후보 자료 선별 루브릭

후보를 **5개 기준**으로 점수화해 합산하고, 상위 N건(기본 5, D8)만 통과시킨다.
매 실행 동일 기준을 적용해 품질을 고정한다.

## 점수 기준 (총 100점)

| 기준 | 배점 | 평가 방법 |
| ---- | ---- | --------- |
| **출처 신뢰도** | 25 | 정부·규제기관·국제기구·학술지(높음) > 주요 언론·산업 보고서(중간) > 블로그·익명·콘텐츠팜(낮음/탈락) |
| **1차 출처 우선** | 20 | 원 보고서·규제 원문·1차 데이터(높음) > 그것을 인용한 2차 보도(낮음). 같은 사실이면 1차를 남기고 2차는 중복 처리 |
| **최신성** | 20 | 최근 6개월 가중(D8). 발효 예정·진행 중 이슈는 가산. 6개월 초과는 감점, 단 "기준 문서"성격(장기 유효)이면 예외 |
| **주제 적합성** | 25 | 요청 주제와의 직접 관련도. 변죽만 울리는 자료는 감점 |
| **고유성(중복 제거)** | 10 | 이미 통과한 자료/`archive/posted-index.md`와 사실·URL 중복이면 0점 처리하여 탈락 |

## 통과 규칙
- 합산 점수 내림차순 정렬 → 상위 **N건** 통과(기본 5, `depth=shallow`→3, `depth=deep`→8).
- **신뢰도 '낮음/탈락' 또는 적합성 현저히 낮음**은 점수와 무관하게 탈락.
- `archive/posted-index.md`에 있는 URL·동일 제목은 무조건 탈락(D6).
- 통과·탈락 각 항목에 **한 줄 사유(`reason`)**를 남긴다(데모 설명·감사 추적용).

## 출력
`curator.json` — `[{...후보, score, reason, passed}]`. 통과 0건이면 그 사실과 사유를 명확히 표기(D19).
